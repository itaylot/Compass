# 🧭 Compass — עוזר טיול משפחתי

אפליקציית ווב-מובייל בעברית (RTL) לניהול טיול משפחתי — מקור אמת אחד ומשותף
במקום אקסל, וואטסאפ וזיכרון. נבנתה במקור לטיול משפחתי של 13 ימים בנורווגיה
(יולי 2026), אבל היא כללית וניתן להזין בה כל טיול.

המשתמשים הם ההורים — לא אנשים טכנולוגיים — ולכן הדגש הוא על פשטות, כפתורים
גדולים, עברית מלאה, וניווט בלחיצה. אפשר להוסיף למסך הבית של iPhone ולהשתמש
כמו אפליקציה (PWA-lite).

## יכולות

- **מסלול לפי ימים** — ציר זמן כרונולוגי, בורר ימים, וכרטיס "הפעולה הבאה" שמראה
  מה עכשיו עם כפתור ניווט גדול. פותח אוטומטית ביום הנוכחי (שעון Europe/Oslo).
- **מפה אינטראקטיבית** — Leaflet + OpenStreetMap, פינים לפי קטגוריה, מסוננים,
  מסונכרנים תמיד עם הרשימה והטבלה.
- **תכנון** — טבלה של כל הטיול עם מיון וסינון.
- **ניווט ב-Waze** — לכל יעד, וגם ליעד חניה נפרד (בערים החניה לרוב לא במקום האטרקציה).
- **הוספה / עריכה / מחיקה** — טופס עם חיפוש מקום (Nominatim) שממלא את המיקום לבד.
  מחיקה רכה עם ביטול (undo).
- **סטטוסים** — מתוכנן / אופציונלי / בוצע. סימון "בוצע" מסונכרן לכל המשפחה.
- **תזכורות יומיות** — רשימה קצרה לכל יום (מעיל גשם, קוד כניסה למלון…).
- **השוואת חלופות לינה**, **מוכנים לטיול?** (בדיקת חוסרים), **מסך חירום**.
- **מזג אוויר אמיתי** מ-[yr.no](https://www.yr.no) דרך ה-Worker (בלי מפתח API).
- **עבודה בלי קליטה** — Service Worker שומר את התוכן שכבר נטען.
- **ייצוא ל-Excel** — קובץ `.xlsx` מסודר (מסלול, לינות, הזמנות, תזכורות).

## בלי מערכת משתמשים — קישור אחד לכולם

אין התחברות ואין תפקידים: **קישור אחד** (`https://<host>/`) שכל מי שיש לו יכול
לצפות, לנווט, לסמן "בוצע", ולערוך/להוסיף/למחוק. הכוונה היא פשטות מקסימלית
למשתמשים לא-טכנולוגיים (משפחה אחת) — הקישור עצמו הוא הפרטיות, אז לא משתפים
אותו בפומבי. מחיקה היא רכה וניתנת לביטול, כך שטעויות קלות לתיקון.

## סטאק

- **React 19 + TanStack Start** (SSR) — רץ כ-**Cloudflare Worker** יחיד.
- **Cloudflare D1** (SQLite) — בסיס נתונים משותף. סכימה ב-`app/migrations/`.
- **Leaflet + OpenStreetMap**, **Nominatim** לחיפוש מקומות, **Waze** לניווט.
- **Tailwind v4**, פונט **Heebo**.
- **exceljs** (נטען דינמית רק בעת ייצוא), **Service Worker** לאופליין.

## מבנה

```
app/
  src/
    routes/                 # TanStack Start routes (SSR)
    components/trip/        # מסכי האפליקציה (Today/Map/Plan/Info + טפסים)
    lib/api/                # פונקציות שרת (createServerFn) — D1, מזג אוויר
    lib/export-xlsx.ts      # ייצוא Excel
    data/trip-items.ts      # מודל נתונים + עזרים
  migrations/               # סכימת D1 (אדיטיבי בלבד)
  public/sw.js              # Service Worker
```

מקור הנתונים כולו עובר דרך `app/src/lib/api/trip.functions.ts` — קל להחליף את
שכבת ה-D1 בכל backend אחר.

## הרצה מקומית

```bash
cd app
npm install
npm run dev        # http://localhost:5173  (ללא D1 — נתונים בזיכרון לצורכי פיתוח)
npm run typecheck
npm run build
```

בפיתוח מקומי אין D1, ולכן פונקציות השרת נופלות ל-store בזיכרון (מתאפס בכל הפעלה)
ומצב הניהול פתוח. בענן, D1 הוא מקור האמת ומפתח הניהול נאכף.

## פריסה (Cloudflare Worker)

הפרויקט נבנה על תבנית אתרים של Higgsfield ונפרס דרכה. הבנייה מפיקה Worker יחיד
(`dist/server/server.js`) + נכסים סטטיים. D1 מופעל דרך `app/app.manifest.json`
(`"db": true`) ומיגרציות מיושמות בפריסה.

> **פרטיות:** אין מנגנון הרשאות — כל מי שיש לו את הקישור יכול לערוך. שמרו את
> הקישור בתוך המשפחה ואל תפרסמו אותו בפומבי.
